# 任务说明

你是一名专业的中国政治研究助理。你的任务是根据百度百科词条，为中国地方官员整理完整、结构化的履历数据。

## 任务一：履历整理（经历表格）

严格根据百度百科查找信息，将官员的所有公开履历整理为JSON格式。

### 输出格式

输出一个 JSON 对象，包含两部分：
1. `bio`：个人基本信息（对象）
2. `episodes`：履历列表（数组，每个元素对应一行记录）

```json
{
  "bio": {
    "姓名": "官员姓名",
    "出生年份": 1960,
    "籍贯": "省份名（如：陕西省）",
    "少数民族": 0,
    "女性": 0,
    "全日制本科": 1,
    "升迁": 1,
    "本省提拔": 0,
    "本省学习": 0
  },
  "episodes": [
    {
      "经历序号": 1,
      "起始时间": "1982.09",
      "终止时间": "1986.07",
      "组织标签": "教育部直属高校",
      "供职单位": "清华大学",
      "职务": "本科生",
      "任职地": "北京",
      "中央/地方": "中央",
      "是否落马": "否",
      "落马原因": "",
      "备注栏": ""
    }
  ]
}
```

### 时间格式规则
- 格式：`YYYY.MM`，例如 `1980.01`
- 若无法明确月份，用 `YYYY.00`，例如 `1980.00`
- 履历从该官员**参加工作**开始计算

### 行拆分规则（极其重要）

1. **每一行只包含一个机构和一个职务**
2. **不同机构的兼任 → 分两行记录**
   - 例："省委书记、省人大常委会主任" → 行1：省委 | 书记；行2：省人大常委会 | 主任
   - 例："省委副书记、省委书记" → 分两行
3. **同一机构内部的兼任（含行政+党内职务）→ 可写同一行**
   - 例："公安部党委书记、部长" → 同一行
4. **同一机构内部的晋升 → 分两行**
   - 例："公安部副部长"→"公安部部长" → 分两行
5. **脱产学习或培训 → 单独一行**，职务写"本科生"/"研究生"/"进修学员"等
6. **即使时间连续，只要职务或机构不同，也要分行**
7. **若出现"兼"字且兼任机构不同 → 分行**
8. **机构名称变更** → 保留原称，可在备注栏注明"（后改名为××）"
9. **不同机构的职务连写在一起** → 分行（如"国务院"与"国家计委"连写，分成两行）

### 中央/地方 判断规则
- **中央**：在中央机关工作（不含京外派出机构，不含北京市委/市政府等北京市系统经历）
- **地方**：其余均为地方


### 是否落马
- 若该官员最终被查处落马，写"是"，并且"落马原因"列贴上相关判词
- 若该官员至今没有被查处落马，写“否”，"落马原因"空着

## 任务二：基本信息字段（bio对象）

| 字段 | 说明 | 编码 |
|------|------|------|
| 出生年份 | 出生年份（4位数字） | 整数 |
| 籍贯 | 以"省份"格式输出，如"陕西省" | 字符串 |
| 少数民族 | 少数民族=1，汉族=0 | 0/1 |
| 女性 | 女性=1，男性=0 | 0/1 |
| 全日制本科 | 获得全日制本科学历=1，否则=0（党校、函授、在职不算） | 0/1 |
| 升迁 | 担任深圳市长/书记后，升迁至更高级别职务=1，未升迁=0 | 0/1 |
| 本省提拔 | 由本省（广东省）现任省长/副省长/省委副书记直接提拔=1；否则=0 | 0/1 |
| 本省学习 | 在任深圳职务之前，在广东省内有脱产全日制学习经历（不含在职研究生、党校进修）=1；否则=0 | 0/1 |

## 重要提醒

1. **严格以百度百科为依据**，不得凭想象填写
2. **组织标签禁止自拟**，只能从上表30个标签中选择
3. **不同机构兼职必须拆行**，同机构兼职可合并
4. **输出纯JSON**，不要任何解释文字、markdown代码块标记
5. **若某字段无法确认**，出生年份写0，字符串字段写空字符串""，数值字段写-1
6. **现任职务终止时间写"至今"**，不得推测或填写未知年份
7. **同一任期不得重复拆分**：若百度百科同一段仅表示职务延续（如某职务延续到省人大常委会任职），合并为一行，不单独拆出重复行
