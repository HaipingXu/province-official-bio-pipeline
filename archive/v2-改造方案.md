## 1. 变更#1：名单改为手动 txt 输入

### 设计目标
彻底废弃 `list_scraper.py` 的 LLM 记忆猜测，由研究者人工维护权威名单。

### txt 文件格式规范

文件路径：`data/{city}_officials.txt`

```
城市：深圳
省份：广东
起始年份：2010

[市长]
许勤, 2010.01-2015.12
陈如桂, 2015.12-2020.01
覃伟中, 2020.01-至今

[市委书记]
王荣, 2005.10-2012.11
马兴瑞, 2012.11-2017.10
王伟中, 2017.10-2021.08
孟凡利, 2021.08-至今
```

**格式规则：**
- 日期格式：`YYYY.MM`，不确定月份写 `YYYY.00`，在任写 `至今`
- 同一人可同时出现在 `[市长]` 和 `[市委书记]` 两节（系统会自动合并）
- 代市长标注：`许勤（代）, 2010.01-2010.08`

### 解析逻辑（`input_parser.py`，新文件）

```python
def parse_officials_txt(path: Path) -> dict:
    """
    返回：
    {
      "city": "深圳",
      "province": "广东",
      "start_year": 2010,
      "mayors": [{"name": "许勤", "start": "2010.01", "end": "2015.12", "acting": False}],
      "secretaries": [...]
    }
    """
```

### 变更影响
- `list_scraper.py`：**停用**（保留文件，不调用）
- `main.py`：Phase 0 改为调用 `input_parser.py`

## 2. 变更#2：提取分两步走

### 问题
当前一次 API 调用要求 LLM 同时完成：
1. 履历时间线提取（事实性，从原文读取）
2. 标签判断（分析性，需要综合推理）

两类任务混在一起导致：分析性字段（升迁/本省提拔）可靠性低，且无法独立核查。

### 新设计：两阶段两 skill 文件

#### Step 2a：职业履历提取（`bio-extraction-step1.md`）

**仅提取事实性信息**，LLM 只需对照原文填写：

```json
{
  "raw_bio": {
    "姓名": "许勤",
    "出生年份": 1961,
    "籍贯": "江苏省",
    "少数民族": 0,
    "女性": 0,
    "全日制本科": 1
  },
  "episodes": [
    {
      "经历序号": 1,
      "原文引用": "1984年参加工作，任深圳市科技局科员",
      "起始时间": "1984.00",
      "终止时间": "1987.00",
      "组织标签": "地方政府",
      "供职单位": "深圳市科技局",
      "职务": "科员",
      "任职地（省）": "广东省",
      "任职地（市）": "深圳市",
      "中央/地方": "地方",
      "是否落马": "否",
      "落马原因": "",
      "备注栏": ""
    }
  ]
}
```

**新增字段**：
- `原文引用`：LLM 从百科原文中摘录该条记录的依据句子（30字内）— 供 battle 对比使用
- `任职地（省）`：完整省份名（如"广东省"）
- `任职地（市）`：完整地级市名（如"深圳市"）；若任职地为省级/中央级无具体市，填""

**不再包含**：`升迁`、`本省提拔`、`本省学习`

#### Step 2b：标签判断 Agent（`bio-labeling.md`，新文件）

输入：Step 2a 完整输出（含所有 episodes）+ 城市、省份信息
任务：专门判断分析性字段

```json
{
  "标签": {
    "升迁": 1,
    "升迁依据": "离任深圳市长后升任黑龙江省委书记（省部级），级别高于副省级",
    "本省提拔": 0,
    "本省提拔依据": "从国家发改委调任深圳，非广东省体系内提拔",
    "本省学习": 0,
    "本省学习依据": "任职深圳前无广东省内脱产全日制学习经历"
  }
}
```

**关键改进**：
- LLM 必须给出 `依据` 字段，无法给出则填 `"无明确依据"`
- `依据` 可被 Qwen 独立核查（判断推理是否合理）
- 人工审查时可直接定位问题

#### 调用顺序

```
百科文本
   │
   ├─→ DeepSeek Step 2a → raw_episodes_ds.json
   │       │
   │       └─→ DeepSeek Step 2b → labels_ds.json
   │
   ├─→ Qwen Step 2a → raw_episodes_qw.json     (并行)
   │       │
   │       └─→ Qwen Step 2b → labels_qw.json
   │
   └─→ Phase 3: 对比 + 裁判
```

## 5. 变更#10：爬虫方法完备方案

### 5.1 当前痛点分析

| 问题 | 根因 |
|------|------|
| 部分 HTML 仅约 763 字 | 百度百科已迁移为 React CSR，履历内容在 JS 执行后才注入 DOM；curl_cffi 拿到首屏 HTML，主体内容区为空 |
| class 名混淆 | Baike 用 CSS Modules，class 后缀随构建变化（如 `J-lemma-content_bq1la`），纯 CSS 选择器不稳定 |
| 403 率偶发反弹 | IP 速率限制 + UA 轮换不够随机 |

### 5.2 推荐策略：三层递进

```
层1（快速）: curl_cffi + 改进解析    → 2-3s/页，命中率 ~70%
层2（可靠）: Playwright + stealth     → 8-15s/页，命中率 ~95%
层3（补充）: 中文维基百科             → 2-3s/页，覆盖率 ~60%
```

### 5.3 层1：curl_cffi + 改进 HTML 解析

**问题所在**：`extract_biography_text()` 用 CSS class 名匹配，但百科 class 已被混淆。

**改进方向**：改用结构性特征和文本特征，而非 class 名：

```python
def extract_biography_text_v2(html: str, name: str) -> str:
    soup = BeautifulSoup(html, "html.parser")

    # 策略1：找所有<p>和<li>，过滤含年份或职务关键词的段落
    # 百科正文段落不管 class 怎么混淆，<p> 标签结构是稳定的
    career_paragraphs = []
    for p in soup.find_all(["p", "li", "td"]):
        text = p.get_text(strip=True)
        if len(text) < 15:
            continue
        # 判断是否为履历段落：含4位年份 或 含职务关键词
        if re.search(r"(19|20)\d{2}", text) or \
           any(kw in text for kw in ["市长","书记","局长","处长","省委","市委","部长"]):
            career_paragraphs.append(text)

    # 策略2：找<table>（百科常用表格呈现履历）
    tables = []
    for table in soup.find_all("table"):
        table_text = table.get_text(separator="\t", strip=True)
        if any(kw in table_text for kw in ["任职","起止","时间","年份"]):
            tables.append(table_text)

    # 策略3：找含官员姓名的<h2>/<h3>章节下的内容
    # （百科页"人物经历"章节）
    for heading in soup.find_all(["h2","h3"]):
        if any(kw in heading.get_text() for kw in ["经历","履历","仕途","职业"]):
            # 收集该 heading 之后直到下一个同级 heading 的所有段落
            ...

    return "\n".join(career_paragraphs + tables)
```

**预期效果**：无需依赖 class 名，按内容特征提取，对百科 React 改版更鲁棒。

### 5.4 层2：Playwright 作为 fallback

**触发条件**：curl_cffi 提取文本 < 2000 字符（说明 JS 内容未加载）

**安装**：
```bash
pip install playwright playwright-stealth
playwright install chromium
```

**实现**：

```python
from playwright.sync_api import sync_playwright
from playwright_stealth import stealth_sync

def playwright_get(url: str, name: str) -> tuple[int, str]:
    """使用 Playwright 获取完整渲染后的百科页面"""
    with sync_playwright() as p:
        browser = p.chromium.launch(
            headless=True,
            args=["--disable-blink-features=AutomationControlled"]
        )
        ctx = browser.new_context(
            user_agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36...",
            viewport={"width": 1920, "height": 1080},
            locale="zh-CN",
        )
        page = ctx.new_page()
        stealth_sync(page)  # 隐藏 navigator.webdriver 等自动化特征

        page.goto(url, wait_until="networkidle", timeout=20000)

        # 等待内容区加载（尝试多个可能的选择器）
        for selector in [
            "div[class*='content']",
            "div[class*='lemma']",
            "article",
        ]:
            try:
                page.wait_for_selector(selector, timeout=5000)
                break
            except:
                continue

        html = page.content()
        browser.close()
        return 200, html
```

**注意事项**：
- Playwright 比 curl_cffi 慢 3-5 倍，仅作 fallback
- 有些反爬系统（如 CAPTCHA）Playwright 也无法绕过，需准备人工 fallback
- Mac/Linux 可直接安装 Chromium；Windows 同样支持

### 5.5 层3：中文维基百科补充

**适用场景**：主要官员（省部级）通常有较完整的维基词条，且维基无反爬限制。

**URL 规律**：
```
https://zh.wikipedia.org/wiki/{name}
```

**优势**：
- 无反爬，`requests` 直接访问
- 履历通常以列表或表格呈现，比百度百科更结构化
- 英文词条（`en.wikipedia.org`）对重要官员有英文履历

**局限**：
- 地级市级别官员（如普通市长）覆盖率约 50%，省委书记等高级别约 85%
- 内容详细度低于百度百科（通常只有任职时间和机构，缺细节）

**用途**：
1. 作为"原文文本"的第二来源，扩充 DeepSeek 的输入
2. 交叉校验时间节点（维基通常有明确的任职起止年月）
3. 作为身份确认的辅助（维基词条通常正确指向对应官员）

**实现**：
```python
def scrape_wikipedia(name: str) -> str:
    url = f"https://zh.wikipedia.org/wiki/{requests.utils.quote(name)}"
    resp = requests.get(url, headers={"Accept-Language": "zh-CN"}, timeout=10)
    if resp.status_code != 200:
        return ""
    soup = BeautifulSoup(resp.text, "html.parser")
    # Wikipedia 结构稳定，直接提取正文
    content = soup.find("div", id="mw-content-text")
    if not content:
        return ""
    return content.get_text(separator="\n", strip=True)[:5000]
```

### 5.6 三层整合逻辑（`bio_scraper_v2.py`）

```python
def scrape_official_v2(name: str, city: str, force: bool = False) -> tuple[bool, str]:
    output_path = OFFICIALS_DIR / f"{name}_biography.txt"

    if output_path.exists() and not force:
        return True, output_path.read_text(encoding="utf-8")

    combined_text = ""

    # --- 层1: curl_cffi ---
    status, html = cffi_get(build_baike_urls(name, city)[0][0])
    if status == 200:
        text_l1 = extract_biography_text_v2(html, name)
        if len(text_l1) >= 2000 and is_correct_person(text_l1, name, city):
            combined_text = text_l1
            print(f"  ✓ 层1 curl_cffi: {len(text_l1)} 字符")

    # --- 层2: Playwright fallback ---
    if len(combined_text) < 2000:
        try:
            _, html_pw = playwright_get(build_baike_urls(name, city)[0][0], name)
            text_l2 = extract_biography_text_v2(html_pw, name)
            if len(text_l2) > len(combined_text):
                combined_text = text_l2
                print(f"  ✓ 层2 Playwright: {len(text_l2)} 字符")
        except Exception as e:
            print(f"  [WARN] Playwright 失败: {e}")

    # --- 层3: 维基百科补充 ---
    wiki_text = scrape_wikipedia(name)
    if wiki_text:
        print(f"  ✓ 层3 维基百科: {len(wiki_text)} 字符（附加）")
        combined_text += f"\n\n=== 维基百科补充 ===\n{wiki_text}"

    if not combined_text or len(combined_text) < 200:
        print(f"  ✗ 所有层均失败")
        print(f"  💡 手动处理：复制百度百科内容 → officials/{name}_biography.txt")
        return False, "all layers failed"

    if not is_correct_person(combined_text, name, city):
        print(f"  ⚠ 身份验证未通过，可能是同名他人")
        return False, "identity check failed"

    output_path.write_text(
        f"官员：{name}\n来源：百度百科+维基百科\n\n{combined_text}",
        encoding="utf-8"
    )
    return True, combined_text
```
