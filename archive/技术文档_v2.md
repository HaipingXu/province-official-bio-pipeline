## 2. Phase 0：人工名单解析（input_parser.py）

### 2.1 设计动机

v1 使用 LLM 自动从百度百科列表页抓取官员名单，存在以下问题：
- 百度百科列表页结构不稳定，经常返回403或缺少信息
- LLM 可能漏掉代市长等特殊任期
- 无法控制任期起止时间的精度

v2 改为**人工维护 txt 文件**，格式简单清晰，完全可控。

### 2.2 文件格式

```
城市：深圳
省份：广东
起始年份：2010
维基列表_市长：https://zh.wikipedia.org/zh-sg/深圳市市长列表

[市长]
许勤, 2010.06-2017.01
覃伟中（代）, 2021.04-2021.05    ← （代）标记代市长
覃伟中, 2021.05-至今             ← 至今表示在任

[市委书记]
王荣, 2010.04-2015.03
```

### 2.3 解析逻辑

- `parse_entry()` 提取姓名、任期起止、是否代理
- `parse_officials_txt()` 读取全部市长+书记，去重合并到 `all_officials`
- `compare_with_wiki()` 可选：抓取维基百科列表页对比，报告差异（不修改名单）

## 3. Phase 1：三层爬取策略（bio_scraper_v2.py）

### 3.1 两层策略（仅百度百科）

| 层 | 工具 | 触发条件 | 预期成功率 |
|----|------|---------|-----------|
| 1 | curl_cffi（Chrome120 TLS伪装）| 默认 | ~70% |
| 2 | Playwright + playwright-stealth | Layer1 文本 < 2000字 | ~92% |

### 3.2 内容提取改进（v2 vs v1）

v1：依赖 CSS class 名（如 `.lemma-summary`），Baidu Baike 改版后失效。

v2：基于**内容启发式**：
- 找所有含年份模式（如 `1990`）或履历关键词（`任职`/`担任`/`书记`）的 `<p>/<li>/<td>` 节点
- 找含任期表头的 `<table>`（如 `任期`/`职务`/`起始`）
- 不依赖任何 class 名

### 3.3 人物核验

`is_correct_person(text, name, city)` 检查提取文本是否确实是目标官员：
- 文本中包含官员姓名
- 文本中包含目标城市名 + 官员职务关键词（市长/书记/主任等）

## 4. Phase 2：DeepSeek两步提取（api_processor_v2.py）

### 4.1 两步架构设计

**v1问题**：一次性提取履历事实+分析标签（升迁/本省提拔/本省学习），LLM容易混淆"描述事实"和"做判断"。

**v2方案**：严格分离两步。

| | Step 1 | Step 2 |
|--|--------|--------|
| 任务 | 提取履历事实 | 分析三个标签 |
| 输入 | 百科原文 | Step1 episodes 列表 |
| 输出 | raw_bio + episodes | 升迁/本省提拔/本省学习（含依据） |
| Prompt | bio-extraction-step1.md | bio-labeling.md |
| 关键字段 | 原文引用（15-50字） | 每标签必须写依据 |

### 4.2 原文引用（`原文引用`字段）

每个 episode 中包含一段原文直接引用（15-50字），作为：
1. **对齐锚点**：Battle 对比时用于匹配 DS 和 QW 提取的同一段经历
2. **可审计性**：人工核查时可直接查证原文

### 4.3 增量保存

每处理完一个官员立即保存结果，防止 API 超时导致全量重跑。

## 5. Phase 3a：Qwen独立核查（verifier_v2.py）

### 5.1 完全独立运行

Qwen 运行相同的 Step1+Step2 提取，完全不看 DeepSeek 的结果，保证验证独立性。

### 5.2 Episode 匹配算法

DS 和 QW 可能对同一人提取出不同数量的 episodes，需要先匹配对应行：

```
相似度 = 原文引用相似度(50%) + 供职单位相似度(30%) + 职务相似度(20%)
阈值 = 0.5，低于阈值视为"无对应行"
```

### 5.3 差异分类

- **field-level diff**：同一行内某字段 DS ≠ QW
- **missing episode**：DS 有但 QW 没有（或反之）
- **label diff**：升迁/本省提拔/本省学习的值不同

所有差异输出到 `logs/diff_report.json`，格式：

```json
[{
  "official_name": "许勤",
  "ds_step1": {...},
  "qw_step1": {...},
  "ds_step2": {...},
  "qw_step2": {...},
  "diffs": [{
    "type": "episode_field",
    "episode_anchor": "1982年9月至1986年7月就读",
    "field": "组织标签",
    "ds_value": "教育部直属高校",
    "qw_value": "地方属高校"
  }]
}]
```

## 6. Phase 3b：Battle表+Claude裁判（battle_generator.py）

### 6.1 裁判策略

- **仅争议字段**调用裁判（DS ≠ QW），避免不必要的 API 调用
- **裁判缓存**：`logs/judge_decisions.json`，重跑时不重复调用
- **降级策略**：ANTHROPIC_API_KEY 未设置 → 自动用 DeepSeek 作裁判

### 6.2 Battle表结构

Sheet 1（履历对比）：

```
| 官员姓名 | 原文引用_DS | DS_起始时间 | DS_组织标签 | DS_供职单位 | DS_职务 | ... |
|          |             | QW_起始时间 | QW_组织标签 | QW_供职单位 | QW_职务 | ... |
|          |             |             |             | 裁判结论    | 裁判理由 | ... |
|          |             |             |             | Final_组织标签 | ...  | ... |
```

- DS 列：灰色 `#E0E0E0`
- QW 列：蓝色 `#CCE5FF`
- 裁判列：紫色 `#E8CCFF`
- Final 列：绿色 `#CCFFCC`
- 争议行整行红色 `#FFCCCC`

Sheet 2（标签对比）：升迁/本省提拔/本省学习三标签的 DS/QW/裁判对比。

## 7. Phase 4：后处理（postprocess_v2.py）

### 7.1 地名标准化

LLM 经常输出省份简称（`广东` 而非 `广东省`），通过 `config.PROVINCE_NORMALIZE` 统一：

```python
"广东" → "广东省"
"北京" → "北京市"
"内蒙古" → "内蒙古自治区"
```

城市名同理通过 `CITY_NORMALIZE` 标准化。

### 7.2 市长/书记标志位判断

**`该条是市长`（AB列）** = 1 当且仅当：
- 职务包含 `市长` 或 `代市长`
- **且**不包含 `副市长`、`常务副`

**`该条是书记`（AC列）** = 1 当且仅当：
- 职务包含 `市委书记`
- **且**不包含 `副书记`、`市委副书记`

**人级标志位**（M/N列）：同一人中任一行 AB=1 → M=1；任一行 AC=1 → N=1。

### 7.3 组织标签校验

不在 30个标签之内的值 → 标记为 `[无效标签]{原值}`，便于人工核查。

### 8.1 三个输出文件

| 文件 | 内容 | 行数参考 |

| `{city}_officials.xlsx` | 全部人员全部经历 | ~300行/城市 |

- 书记表：目标行蓝色高亮
- 冻结首行+前4列（E列=姓名 为右边界）

## 9. 提取规则（单一信息源）

### 9.1 bio-extraction-step1.md

Step1 相较 v1（bio-extraction.md）的变化：

| 变化项 | v1 | v2 (Step1) |
|--------|-----|-----------|
| `原文引用`字段 | 无 | 每条必须，15-50字 |
| 任职地 | 单列 `任职地` | 拆分为 `任职地（省）` + `任职地（市）` |
| 地名格式 | 无强制规定 | 必须使用完整正式名称（广东省、深圳市等） |
| 兼职处理 | 允许合并 | 不同机构兼职强制分行 |
| 副市长/代市长 | 可同行 | 必须分行 |
| 升迁/本省提拔/本省学习 | 包含 | **不包含**（移至Step2） |

### 9.2 bio-labeling.md

Step2 输入为完整 episodes 列表，输出三标签及依据：

```json
{
  "升迁": 1,
  "升迁依据": "离任深圳市委书记后调任黑龙江省委书记，由副省级升至正省部级",
  "本省提拔": 0,
  "本省提拔依据": "从国家发展改革委（中央）调任深圳，非广东省内晋升",
  "本省学习": 0,
  "本省学习依据": "履历中在广东省内无脱产全日制学习记录"
}
```

规则要点：
- **升迁**：基准为副省级→省部级晋升（深圳书记/市长均为副省级）
- **本省提拔**：看上一条经历，须为省级领导机构（非其他省、非中央）
- **本省学习**：须为脱产全日制（不含党校进修、在职研究生）
- **-1**：无法判断时填-1，依据字段写"无法从履历判断"

## 10. 列架构（A→AC）

| 列 | 名称 | 变化 |
|----|------|------|

| AC | 该条是书记 | 后移（原AA） |
| — | 厅局级及以下 | **删除**（原I列，经讨论移除） |
